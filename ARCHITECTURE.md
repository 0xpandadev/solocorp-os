## Agent-under-Agent（入れ子構造）の詳細

各軍団は**人間（CEO）を通さずに内部でループ**を完結させる。

### 例：クリエイター軍団の内部会議
```
CEO: 「今のAIトレンドでバズる記事を書いて」
        ↓
Hub: クリエイター軍団を召喚
        ↓
  [内部会議 START — CEOには見えない]
  ①トレンド分析師: 「今週のAIバズワードはXXX。ターゲットは起業家層」
  ②コピーライター: 「<初稿作成>」
  ③毒舌編集長: 「凡庸すぎる。フックが弱い。リード文を3回書き直せ」
  ②コピーライター: 「<改稿v2>」
  ③毒舌編集長: 「まだAIっぽい。具体的な数字を入れろ」
  ②コピーライター: 「<改稿v3>」
  ③毒舌編集長: 「OK、これは本物のエッジがある」
  [内部会議 END]
        ↓
Hub: CEOに最終完成品を提出
```

## 記憶システム（Context Preservation）

```
notes/
├── INTENT.md          ← CEOの「魂（Why）」を記録。常に参照する
├── investments/
│   └── MEMORY.md      ← 投資に関する過去の分析・決定事項
├── consulting/
│   └── MEMORY.md      ← コンサル案件の文脈・フレームワーク
├── media/
│   └── MEMORY.md      ← SNSペルソナ・トーン&マナー
└── dev/
    └── MEMORY.md      ← 技術設計・禁止された実装・環境情報
```

## 人間の役割（The Last 1%）

| やること | やらないこと |
|---|---|
| WhyとVibeを語る | Howを細かく指示する |
| 最終成果物を承認する | 途中のレビューに介入する |
| 倫理的な防波堤となる | AIの内部会議に口を出す |
| 新しい事業の意図をINTENT.mdに書く | 技術的なルートを決める |
